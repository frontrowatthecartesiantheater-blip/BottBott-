Where Agents Apply
Eight functions worth exploring

Every one of these compresses knowledge work — pushing tasks to 70–80% completion before a principal's attention is required. Mark the ones that feel relevant in the form below.

01

Research Agent

First-pass research on CRA interpretation bulletins, ITA provisions, case law, and technical interpretations — surfaced before you open a document. Cedar is regularly doing original research on issues other accountants can't resolve. An agent that handles the first sweep compresses that from hours to minutes.

02

Step Plan & Memo Drafting

Reorganizations and M&A transactions require detailed step plans — precise sequences of legal and tax steps. These follow logical patterns but drafting from scratch is time-consuming. An agent that takes the desired outcome and corporate structure as inputs and produces a first-draft step plan extends the proposal generator you've already built into the delivery phase.

Tax Legal
03

Calculation Workflows

ACB tracking, paid-up capital computations, safe income determination, RDTOH, CDA balances, purchase price allocation — complex but rule-based. An agent that takes inputs, runs the calculation logic, and produces a formatted report eliminates both manual calculation time and the review burden that follows it.

Tax Valuations
04

Precedent & Institutional Memory

Cedar has years of accumulated work — prior structures, positions taken, memos written, issues resolved. Right now that knowledge likely lives in folders and in the principals' heads. A local agent that indexes Cedar's own work product and lets anyone on the team query it turns your firm's history into a searchable asset.

05

CRA Audit Response Drafting

Audit defense follows a consistent structure: facts, issue, CRA's position, taxpayer's position, argument, conclusion. An agent that takes the audit issue and client facts as inputs and produces a first-draft submission compresses what is often a full day of drafting into a review and refinement task.

Due Diligence Assistant

In M&A work Cedar reviews financial statements, corporate documents, and contracts to identify tax exposures and risks. Deployment & Security
Two practical paths for Cedar

There are several approaches to deploying AI without client data leaving your network. Based on Cedar's existing Microsoft infrastructure (Outlook, Office 365), these two are the most practical fit. Alternative paths exist for Google-native environments.

Path 01

Local Deployment

An open-source model runs entirely on Cedar's own hardware — a dedicated workstation or internal server. No data leaves the building under any circumstances. Tools like Ollama make this accessible without significant infrastructure overhead.

Data security Complete — nothing leaves the network
Capability Good — lower than frontier models, workable for most functions listed above
Infrastructure A capable workstation or dedicated server on-premise
Best for Highest-sensitivity work; maximum control
Path 02

Azure Private Endpoints

Frontier-level models run within Cedar's own Azure cloud tenant via private endpoints. Data never touches Microsoft's shared infrastructure. Cedar is already in the Microsoft ecosystem — this extends that environment rather than introducing a new one.

Data security Strong — data stays within Cedar's Azure tenant
Capability Full frontier model quality (GPT-4o class)
Infrastructure Azure subscription; Microsoft contractual data residency
Best for Maximum capability with enterprise-grade compliance

Note: These recommendations are based on Cedar's Microsoft stack as described in the Missive case study. If Cedar has since moved to or added Google Workspace, a Google Vertex AI deployment within your existing Google environment is an equally strong alternative and worth discussing.
